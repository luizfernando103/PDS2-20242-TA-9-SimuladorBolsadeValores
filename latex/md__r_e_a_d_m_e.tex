{\ttfamily PROGRAMAÇÃO E DESENVOLVIMENTO DE SOFTWARE II}

Este projeto se baseia na criação de um simulador de bolsa de valores, uma plataforma virtual que possibilita aos usuários praticarem investimentos financeiros sem arriscar dinheiro real. Ele replica as condições reais do mercado financeiro, oferencendo ferramentas e recursos semelhantes aos que são encontrados em plataformas de negociações reais. Os usuários poderão comprar e vender ações com dinheiro fictício enquanto acompanham o desempenho de seus investimentos. Esse simulador é útil para investidores iniciantes aprenderem sobre o mercado financeiro, experimentarem diferentes estratégias de investimento e ganharem confiança antes de investirem com dinheiro real.\hypertarget{md__r_e_a_d_m_e_autotoc_md1}{}\doxysection{\texorpdfstring{Integrantes}{Integrantes}}\label{md__r_e_a_d_m_e_autotoc_md1}

Ao digitar “d” no menu principal, você será redirecionado para o menu do administrador, nele haverá diversas opções para o usuário do programa ter a capacidade de alterar os dados do programa livremente\+:


\begin{DoxyItemize}

\item {\bfseries{Remover ativo de ação}}\+: Digitando “r”, pode-\/se remover um ativo de ação o qual for selecionado do mercado virtual
\item {\bfseries{Cadastrar novo usuário}}\+: Digitando “c”, pode-\/se cadastrar um novo usuário ao sistema do programa, informando seu nome e seu primeiro depósito em sua carteira
\item {\bfseries{Remover conta de usuário}}\+: Digitando “e”, pode-\/se remover um usuário cadastrado o qual foi selecionado do sistema

Caso o usuário queira alterar os dados iniciais do programa, ele pode utilizar o arquivo “dados.\+txt” localizado no próprio diretório do programa. Esse arquivo contém todos os dados os quais o software terá ao inicializar. O arquivo é dividido em três partes. A primeira está relacionada aos ativos do mercado virtual e pode-\/se alterá la baseando-\/se na seguinte estrutura a qual ela segue\+:

\+\_\+\+Número de ativos diferentes ~\newline

 …\+\_\+

A segunda parte diz sobre os usuários cadastrados no sistema e suas respectivas carteiras. Ela segue a seguinte estrutura\+:

\+\_\+\+Número de usuários ~\newline

 …\+\_\+

A terceira parte contém as transações que ocorreram antes da inicialização do programa e possui a seguinte estrutura\+:

\+\_\+\+Número de transações ~\newline

 …\+\_\+

Cada seção do arquivo é dividida por uma linha vazia, assim como cada usuário da segunda parte.\hypertarget{md__r_e_a_d_m_e_autotoc_md15}{}\doxysection{\texorpdfstring{Principais Dificuldades}{Principais Dificuldades}}\label{md__r_e_a_d_m_e_autotoc_md15}
A principal dificuldade encontrada no projeto foi relacionada ao arquivo “main.\+cpp”, que estava ficando muito grande e confuso. Como resolução, tivemos que criar os arquivos “simulation.\+hpp” e “simulation.\+cpp” para manter uma programação mais organizada e menos confusa, além de fazer pequenas alterações nos outros arquivos que já estavam prontos.
